# プロジェクト構造の説明

## 2024-01-02 - すべてのディレクトリの役割

### 質問
もっと簡単にすべてのディレクトリのそれぞれの役割をしりたい！できるだけ噛み砕いて教えて欲しい

### 回答
プロジェクトの各ディレクトリの役割を、できるだけ分かりやすく説明します。

## プロジェクト全体のイメージ

**家を建てるイメージで考えると：**

```
🏠 プロジェクト（家全体）
├── 📁 app/          ← 家の骨組み・電気・水道（仕組み）
├── 📁 templates/    ← 壁紙・内装（見た目）
├── 📁 static/       ← 家具・装飾（飾り付け）
├── 📁 tests/        ← 点検・検査（テスト）
├── 📁 Chat/         ← メモ帳（学習記録）
├── 📁 Docs/         ← 設計図（要件定義）
└── 📄 tasks.db      ← 倉庫（データ保存）
```

## 各ディレクトリの役割（超簡単に）

### 📁 app/ - アプリの仕組み（最重要！）

**役割：アプリの動く仕組みを書く場所**

```
app/
├── main.py       ← アプリの入口（玄関）
├── models.py     ← データの形を定義（設計図）
├── database.py   ← データベースとのやり取り（倉庫の管理）
└── routers/      ← APIのルート（道案内）
```

**具体例：**
- `main.py`: 「タスクを追加して」というリクエストを受け取る
- `models.py`: タスクは「タイトル、説明、象限」が必要と定義
- `database.py`: データベースにタスクを保存する
- `routers/`: `/api/tasks` という道を作る

**イメージ：**
- 家の骨組みや電気・水道の配線
- 見えないけど、全部ここで動いている

---

### 📁 templates/ - 見た目（HTML）

**役割：ユーザーが見る画面を作る場所**

```
templates/
└── index.html    ← メイン画面（後で作る）
```

**具体例：**
- タスク一覧を表示する画面
- タスクを追加するフォーム
- 4象限のマトリックス表示

**イメージ：**
- 家の壁紙や内装
- ユーザーが実際に見る部分

---

### 📁 static/ - 飾り付け（CSS、JavaScript）

**役割：見た目をキレイにする、動きを追加する**

```
static/
├── style.css     ← デザイン（後で作る）
└── script.js     ← 動き（後で作る、HTMX、Sortable.jsなど）
```

**具体例：**
- ボタンの色、レイアウト
- HTMX、Sortable.jsなどのライブラリ

**イメージ：**
- 家具や装飾品
- 見た目をキレイにする、動きを追加する

---

### 📁 tests/ - テストコード

**役割：アプリが正しく動くか確認する**

```
tests/
└── test_tasks.py  ← タスク機能のテスト（後で作る）
```

**具体例：**
- タスクが正しく追加できるか
- タスクが正しく削除できるか

**イメージ：**
- 家の点検・検査
- 壊れていないか確認する

---

### 📁 Chat/ - 学習メモ

**役割：質問と回答を記録する**

```
Chat/
├── HTMX.md
├── FastAPI.md
├── Python.md
└── ...
```

**具体例：**
- 「HTMXとは？」という質問と回答
- 「仮想環境とは？」という質問と回答

**イメージ：**
- 学習ノート
- 後で見返せるように記録

---

### 📁 Docs/ - 設計図・要件定義

**役割：プロジェクトの設計書**

```
Docs/
├── 要件定義書.md
├── 技術スタック比較.md
└── ...
```

**具体例：**
- どんな機能が必要か
- どんな技術を使うか

**イメージ：**
- 家の設計図
- プロジェクトの全体像

---

### 📄 tasks.db - データベースファイル

**役割：データを保存するファイル**

**具体例：**
- タスクのデータ（タイトル、説明、象限など）
- SQLite形式のファイル

**イメージ：**
- 倉庫
- データを保存しておく場所

---

### 📁 venv/ - 仮想環境（触らない）

**役割：Pythonのパッケージを入れる場所**

**注意：**
- 基本的に触らない
- Gitにもコミットしない（.gitignoreに追加）

**イメージ：**
- 道具箱
- 必要な道具（パッケージ）を入れておく

---

## 開発の流れ（どのディレクトリから始める？）

### バックエンドから組み立てる（このプロジェクトの方針）

```
1. app/database.py    ← まずデータベース
2. app/models.py      ← 次にデータモデル
3. app/main.py        ← 次にAPI
4. templates/         ← 最後に画面
5. static/            ← 最後にデザイン
```

**イメージ：**
- 家を建てる順番
- 骨組み → 内装 → 装飾

---

## 簡単なまとめ

| ディレクトリ | 役割 | イメージ |
|------------|------|---------|
| **app/** | アプリの仕組み | 家の骨組み |
| **templates/** | 見た目（HTML） | 壁紙・内装 |
| **static/** | 飾り付け（CSS/JS） | 家具・装飾 |
| **tests/** | テスト | 点検・検査 |
| **Chat/** | 学習メモ | メモ帳 |
| **Docs/** | 設計図 | 設計図 |
| **tasks.db** | データ保存 | 倉庫 |
| **venv/** | パッケージ | 道具箱（触らない） |

---

## 重要なポイント

1. **app/** が最重要
   - アプリの動く仕組みが全部ここ

2. **templates/** と **static/** は見た目
   - ユーザーが見る部分

3. **バックエンドから組み立てる**
   - app/ → templates/ → static/ の順

4. **Chat/** と **Docs/** は学習・記録用
   - 開発には直接関係ないが、重要
